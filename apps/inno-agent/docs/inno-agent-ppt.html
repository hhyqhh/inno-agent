Overview

项目目标

🎯 学习者画像

持续理解用户的目标、能力状态、误区、偏好和复习节奏，让每次回答都量身定制

📚 知识图谱归档

接入 Graphify，将学习内容、笔记、项目知识归档为可查询的 Wiki / Graph 知识库

💬 多渠道接入

支持飞书，后续扩展 QQ、微信、企业微信，统一接口，无缝切换

⏰ 定时任务

学习复盘、知识库更新、画像反思、间隔复习提醒，后台自动运行

核心原则：不修改 Pi 内核，优先通过 SDK、Extension、Custom Tools 组合能力

Memory System

三层记忆系统

🧠 L1 — 学习者画像记忆（LearnerProfile）

保存目标、知识状态、误区、学习行为、自我调节、动机情绪、偏好与证据。
证据驱动 · 可解释 · 可修正 · 时间敏感 · 回答"这个学习者是谁、怎么学"

📚 L2 — Graphify Wiki 知识库

直接接入 Graphify，归档学习笔记、知识总结、概念关系、项目文档。
回答"学过什么知识、知识间有什么关系"

💬 L3 — Pi 会话记录（SessionManager）

由 Pi 原生 SessionManager 管理：对话历史、工具调用记录、会话恢复、上下文压缩。
回答"最近聊了什么、Agent 做过什么"

L1 存能力判断　|　 L2 存知识内容　|　 L3 存近期上下文

Scheduler

定时任务系统

  • 📝
    daily_review
    每天总结学习事件，更新画像，生成明日建议
  • 📊
    weekly_summary
    目标进展、知识状态变化、活跃误区、下周计划
  • 🗂️
    graphify_update
    定时扫描 input，更新 Graphify Wiki / Graph
  • 🔁
    spaced_review
    找出 review_due_at 到期概念，推送间隔复习
  • 🔔
    push_reminder
    自定义提醒，推送到指定渠道

用户自然语言创建

"每天晚上 10 点帮我总结 今天学了什么，发到飞书" ↓ Agent 调用 ↓ create_scheduled_job({ name: "daily_review", cron: "0 22 * * *", taskType: "daily_review", channel: "feishu" })

⚡ 作为独立后台服务运行，不依赖 CLI 进程常驻
