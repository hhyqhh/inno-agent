# Review Planner

把"学过的东西"转成"能在合适时间点被重新激活"的卡片和日程,对抗遗忘。

## 何时触发

用户已经有一批错题或知识点,需要:
- 生成可反复刷的复习卡(尤其 Anki 导入格式)
- 排出科学的复习时间表(而非"明天再复习一遍")

## 艾宾浩斯间隔参数

本技能采用以下复习间隔(基于艾宾浩斯遗忘曲线的常用简化版):

| 轮次 | 距上次复习 | 说明 |
|---|---|---|
| 第 1 次 | 当天 | 首次学习,立即生成卡片 |
| 第 2 次 | +1 天 | 次日复习,遗忘最快阶段 |
| 第 3 次 | +3 天 | 短期巩固 |
| 第 4 次 | +7 天 | 一周节点 |
| 第 5 次 | +16 天 | 中期巩固 |
| 第 6 次 | +35 天 | 长期巩固,之后视为已掌握 |

> 注:日期无法从对话内精确计算时,以"今天"为第 1 次的锚点,只标注相对天数(如 +1d / +3d / +7d / +16d / +35d),让用户自己落到日历。

## 复习卡格式(Anki 兼容)

写入 `cards/<主题>.csv`,**文件头三行固定**:

```
#separator:Semicolon
#html:false
正面;背面;标签
```

之后每行一张卡,字段用 `;` 分隔:
```
什么是空间推理;指理解物体间位置/方向/距离等空间关系的能力;VLM,空间推理,概念
VLM空间推理的主要挑战;1)语言偏差 2)细粒度定位弱 3)3D→2D投影模糊 4)参照系混淆;VLM,空间推理,挑战
```

**卡片设计原则**:
- **一面一知识点**:不要把整页笔记塞进一张卡。一张卡只问一个明确问题。
- **正面尽量短**:能一眼看懂要回忆什么。
- **背面给最小够用的答案**:定义 + 一个例子/对比,不要长篇。
- **标签用于分类检索**:固定含主题标签(如 `VLM`)+ 类型标签(如 `概念`/`错因`/`步骤`)。
- 单次最多生成 20 张;超过的拆成多个主题文件。

## 三种卡片模板

| 类型 | 正面 | 背面 | 适用 |
|---|---|---|---|
| 概念卡 | 术语 / "X 是什么" | 定义 + 例子 | 定义、原理 |
| 错因卡 | "典型错误:X" | 正确做法 + 为什么错 | 易错点、思维陷阱 |
| 步骤卡 | "解 X 类题的步骤" | 编号步骤 + When-X-then-Y 规则 | 程序性知识、解题流程 |

## 复习日程表格式

写入 `review-schedule.md`,用相对天数(精确日期由用户落地):

```markdown
# 间隔复习日程

> 以首次学习日为 D0,按艾宾浩斯曲线间隔复习。

| 卡片集 | D0(当天) | D+1 | D+4 | D+11 | D+27 | D+62 |
|---|---|---|---|---|---|---|
| VLM空间推理 | ✅ 生成12张 | 复习全部 | 复习错过的 | 复习仍错的 | 抽查 | 终测 |
| 高数-极限 | ✅ 生成8张 | 复习全部 | ... | ... | ... | ... |

## 使用方法
1. 把 `cards/*.csv` 导入 Anki(File → Import,分隔符选 Semicolon)
2. 按上表日期打开 Anki 过一遍当天该复习的牌组
3. 第 6 次仍答对的卡可从复习池移除(已掌握)
```

## 工作流程

1. **清点知识点**:从用户的错题/笔记里提取可制卡的知识点清单。
2. **选卡类型**:每个知识点判断用概念卡 / 错因卡 / 步骤卡哪种。
3. **写卡**:按格式生成,每张遵守"一面一知识点、正面短、背面最小够用"。
4. **分组**:按学科或主题把卡片分到不同 `<主题>.csv` 文件。
5. **排程**:为每个卡片集生成一行复习日程,套用 6 轮间隔。
6. **导出**:写 `cards/*.csv` + `review-schedule.md` 到工作区,告知用户卡片数、文件路径、Anki 导入方法。

## 原则

- **少而精**:20 张高质量卡 > 100 张流水账卡。只把真正需要记住的做成卡。
- **可离线**:产出的是文件,用户脱离对话也能用。
- **不编造知识点**:卡片内容必须基于用户实际提供的错题/材料,不要为了凑数杜撰考点。
