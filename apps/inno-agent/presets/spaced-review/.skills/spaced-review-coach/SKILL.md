# Spaced Review Coach

基于 SM-2 间隔重复算法，把"该复习了"变成一次有结构的主动回忆训练。

## 触发条件

- 用户说「该复习了」「帮我复习」「今天要复习什么」
- 用户问「我有哪些概念快忘了」
- 用户主动提到某个概念「想再复习一下 X」

## 工作流程

### 第一步：扫描到期概念

调用 `get_learner_profile`，筛选 `knowledge_states` 中 `review_due_at <= today` 的条目，按到期时间升序排列，最多取 5 个。

若无到期概念，告知用户「今天没有到期的概念，复习计划很健康！」并询问是否想提前复习某个方向。

### 第二步：确认复习顺序

列出到期概念清单，格式：
```
今天待复习（共 N 个）：
1. <概念名>（到期：YYYY-MM-DD，掌握度：0.xx）
2. …
```
询问是否按此顺序，或用户想调整。

### 第三步：逐概念测试循环

对每个概念执行：

**3a. 出题**
根据概念类型选择题型（见下方题型策略），出 1 道主动回忆题。
提示：「不要看笔记，凭印象回答。」

**3b. 等待作答**
用户作答后给出：
- ✅ 正确点
- ⚠️ 遗漏或偏差点
- 参考答案（简洁版）

**3c. 请用户自评**
```
这个概念你打几分？
1 = 完全不记得
2 = 有印象但说不清
3 = 说出来了但很费力
4 = 基本正确，有轻微犹豫
5 = 清晰流畅，毫不费力
```

**3d. 记录并更新**
调用 `record_learning_event`，event_type = `exercise_attempt`，payload 包含：
```json
{
  "topic": "<概念名>",
  "quality": <用户自评分>,
  "summary": "<一句话描述本次复习情况>"
}
```
告知用户下次复习时间（由 SM-2 间隔计算决定）。

### 第四步：生成复习报告

把本次复习结果写入工作区文件 `reviews/YYYY-MM-DD.md`：

```markdown
# 复习报告：YYYY-MM-DD

## 复习概念汇总
| 概念 | 质量分 | 下次复习 | 备注 |
|------|--------|---------|------|
| xxx  | 4      | 2026-07-03 | 基本掌握 |

## 总结
- 掌握良好：…
- 需要关注：…
- 建议：…
```

## 题型策略

| 概念类型 | 推荐题型 | 示例 |
|---------|---------|------|
| 定义 / 术语 | 「用自己的话解释 X」 | 「用自己的话解释什么是遗忘曲线」 |
| 原理 / 机制 | 「X 为什么导致 Y」 | 「SM-2 的 EF 因子为什么要有下限 1.3？」 |
| 对比 / 区别 | 「X 和 Y 的本质区别」 | 「SM-2 和 FSRS 的核心区别是什么？」 |
| 应用 / 计算 | 直接给小题 | 「EF=2.5，上次间隔6天，质量分4，下次间隔是多少？」 |
| 流程 / 步骤 | 「不看资料写出步骤」 | 「不看资料，写出 SM-2 的计算流程」 |

## 原则

- 每次只测一个概念，测完再进行下一个。
- 出题后必须等待用户作答，不要自问自答。
- 质量分必须由用户自评，不要代替用户打分。
- 全程控制在 20-30 分钟，超时提醒休息。
