# Concept Bridge

把新概念和已学知识编织在一起，生成对比、巩固和组合练习。

## 工作流程

1. **扫描关联概念**：从 L1 上下文或 `get_learner_context` 获取 `knowledge_states[]`，筛选规则：
   - domain 相同或父子关系（如 `python.pandas` 和 `python` 视为同域）
   - mastery >= 0.3（太低表明用户对该概念本身尚未建立稳定理解，不适合作为桥梁锚点）
   - 按 mastery 降序排列，取前 3 个
2. **检索知识内容**：对每个关联概念调用 `l2_query(概念名)` 获取 Wiki 页面内容；若无结果则仅基于 L1 的诊断信息生成关联。
3. **生成对比表**：对每个关联概念，按以下模板产出：

```
| 对比维度 | {新概念} | {关联概念} |
|---------|---------|-----------|
| 核心定义 | ...     | ...       |
| 使用场景 | ...     | ...       |
| 关键差异 | ...     | ...       |
| 易混点   | ...     | ...       |
| 关联启示 | ...     | ...       |
```

4. **生成桥接练习**：
   - **热身题**（1 道）：纯用关联概念的知识，让用户先进入状态
   - **巩固题**（1 道）：纯用新概念的知识，检验刚学的内容
   - **组合题**（1 道）：同时需要新旧概念才能解决，在应用中分辨何时用哪个
5. **写入工作区**：使用 `create_practice_lab` 创建练习文件，`suggestedCommand` 填写运行命令。工作区文件路径统一为：
   - `practice/bridge_<新概念名>_<YYYY-MM-DD>.<ext>`
6. **归档**：对比表和桥接练习调用 `l2_archive` 归档到 L2，tags 必须包含新概念名和所有关联概念名，origin 设为 `agent_inferred`。标题格式：`[螺旋回顾] {新概念名} 关联 {关联概念名}`。

## 练习文件格式

以 Python 为例，其他语言同理调整：

```python
# 螺旋回顾 - 桥接练习: {新概念名} × {关联概念名}
# 日期: {YYYY-MM-DD}

# ===== 热身题: {关联概念名} =====
# 题目: ...

# ===== 巩固题: {新概念名} =====
# 题目: ...

# ===== 组合题: {新概念名} × {关联概念名} =====
# 题目: ...

if __name__ == "__main__":
    # 测试用例
    pass
```

## 原则

- **锚定已知**：关联概念必须来自 L1 画像的真实数据，不能凭空编造。
- **对比优于堆砌**：对比表是核心产出，练习是对比的应用检验，两者同等重要。
- **题目场景化**：组合题需要给一个贴近实际的场景，而不是干巴巴的"请同时使用 A 和 B"。
- **降级兜底**：若 L2 中无关联概念的 Wiki 页面，用 L1 的 mastery/diagnosis 数据仍可生成对比；若 L1 中无同域概念，告知用户"这是该领域的第一个概念，暂时没有已学关联知识，正常讲解"。
