# Spiral Review

根据概念掌握度和时间间隔，生成递进式螺旋回顾题目。

## 触发条件

收到 `spaced_review` 类型的调度任务，且任务 prompt 中包含概念及其关联概念信息（由 `concept-bridge` 在工作流中归档）。

## 三级回顾规则

### Level 1 — 关联回顾（概念学习当天）

- **目标**：巩固新旧概念的区别和联系
- **内容**：重新展示对比表 + 1 道类比判断题
- **题型**：给出一个具体场景，要求判断"应该用 {新概念} 还是 {关联概念}"，并解释原因
- **难度上限**：不引入新题型，仅在已讲过的范围内巩固

### Level 2 — 组合回顾（概念学习 3 天后）

- **目标**：验证能否在复杂场景中区分和综合运用
- **内容**：1 道需要同时运用新旧两个概念的编程题或分析题
- **题型**：给定一个多步骤问题，其中至少一步需要关联概念、至少一步需要新概念
- **降级条件**：若当前 mastery < 0.5，降级为 L1 难度（再出一道类比判断题 + 一道简单巩固题）
- **升级条件**：若当前 mastery >= 0.85，题目增加第 3 个关联概念作为干扰项

### Level 3 — 迁移回顾（概念学习 7 天后）

- **目标**：验证能否将新旧概念综合运用到陌生场景
- **内容**：1 道开放设计题
- **题型**：要求设计解决方案，而非单纯实现功能。题目描述一个不熟悉的场景，用户需要自行判断该用哪些概念、如何组合
- **降级条件**：若当前 mastery < 0.6，降级为 L2 难度（给一道有明确步骤提示的组合题）

## 回顾后评估

每级回顾结束后必须执行：

1. **记录事件**：调用 `record_learning_event`（event_type: `exercise_attempt`，context.concept_ids 包含新旧概念）
2. **更新掌握度**：

| 结果 | 操作 |
|------|------|
| 答对且思路清晰 | `mastery_delta = +0.05`（L1）/ `+0.08`（L2）/ `+0.12`（L3）|
| 答对但犹豫或思路模糊 | `mastery_delta = +0.02`，不分级别，`diagnosis` 标注"概念边界模糊，需要更多对比练习" |
| 答错且属于新旧混淆 | 调用 `record_learning_event`，在 `derived_signals.misconception_candidates` 中写明混淆类型和描述 |
| 答错且属于概念遗忘 | `mastery_delta = -0.05`，`diagnosis` 标注"间隔过长导致遗忘，建议缩短复习周期" |

3. **调整复习周期**：回顾结果将自动影响 L1 的 `stability` 和下一次 `review_due_at`（由 `auto-profile.ts` 自动计算）

## 与调度器的衔接

- 不需要自己创建调度任务 —— 已由系统的自动间隔复习功能（`auto-review.ts`）处理
- 收到调度任务后，按当前 mastery 决定回顾级别和难度
- 回顾完成后不手动设 `review_due_at`，交由 L1 自动计算下一个复习时间点

## 原则

- **递进不重复**：L1/L2/L3 是三级递进关系，不是三道同类题。每一级考察的是不同深度的能力。
- **降级保底**：掌握度不够时自动降级，不强行推难度。挫败比枯燥更容易导致放弃。
- **混淆即信号**：新旧概念混淆是最有价值的诊断信息，必须记录为 `misconception_candidates` 而非简单判错。
- **稳定期后放手**：一个概念连续两次 L3 迁移题都答对且 mastery >= 0.9，在 `diagnosis` 中标注"该概念已达稳定迁移水平，转入长期记忆维护"，后续仅参与被动复习而非主动螺旋。
