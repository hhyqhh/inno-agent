# 班级试卷讲评

把一次考试的试卷、答案与班级答题数据，整理成一份可直接走进课堂使用的讲评方案与配套材料。不按题号从头讲到尾，而是按讲评价值排序、聚焦班级真实问题。

## 工作前提

教师需要提供以下三类材料，缺一不可（答案可与试卷合一）：

1. **试卷文件**：PDF、Word 或扫描图片。
2. **标准答案与评分标准**：可单独上传，也可附在试卷文件之后。主观题需要列出得分点。
3. **班级作答明细**：优先 Excel。最低要求是每名学生每题的作答（客观题选项、主观题得分）与总分。格式见 [输入格式](references/input-schema.md)。

第一版不要求识别全班手写答题卡。教师若只交了纸质答题卡，应提示其先用阅卷系统导出明细表再上传。

## 工作流程

1. **读取材料**。读取试卷、答案与作答明细。客观题与主观题分别处理；图片试卷先做文字识别再校对，不得补写资料中不存在的题干或答案。无法确认的题目标为待确认。
2. **解析试卷结构**。按 [试卷结构](references/exam-structure.md) 抽取每题：题号、题型、分值、标准答案、知识点、难度、题目关联、主观题评分点。
3. **请教师确认一次结构**。以简短清单列出：题量、各题型与分值、知识点分布、存疑项。题号或答案错位会污染全部后续分析，**教师确认前不得进入统计**。
4. **匹配作答数据并统计**。按 [统计口径](references/statistics.md) 计算：每题正确率/平均得分、客观题选项分布、主观题得分段、知识点得分率、总分分布（均分、最高/最低、各分数段人数）。
5. **诊断错误类型**。不只报“第 X 题错误率高”，还要给出可能原因。客观题列高频错误选项及混淆点；主观题按 [主观题错误归因表](references/statistics.md#主观题错误归因表) 归类（公式选错、漏步骤、计算错、条件用不全、表达不规范、前置知识缺失）。
6. **排讲评优先级**。按 [讲评价值公式](references/review-formula.md) 给每题一个讲评价值分，再分为四档：必须重点讲 / 简要讲解 / 学生自行订正 / 无需统一讲解。
7. **产出讲评方案**。对重点题逐题写讲评稿（见 [讲评稿模板](references/review-script.md)）：班级表现、讲评目标、建议讲评过程、板书内容。典型错误展示需把教师上传的学生答案匿名化。
8. **生成补偿练习**。针对班级问题出题，每题标注对应原试卷题号与知识点，覆盖：课堂即时检测、同类巩固、易错点辨析、课后补偿、提高题。不得直接照搬原题。
9. **生成个人订正单**。按 [订正单模板](references/student-report.md) 为每名学生给出重点订正题、主要问题、建议任务量。
10. **汇总并落地**。把全部产物写入工作区目录（见 [交付物结构](references/deliverables.md)），并在对话中给出班级总体结论与课堂使用建议。

## 内容质量规则

- 所有正确率、得分率、分布数字必须由作答明细计算得出，可追溯到原始数据；不得编造或“估算”具体百分比。
- 知识点与难度若资料未给出，标为“待教师补充”，不要凭空填写；后续公式中涉及的权重缺项时按默认值处理并注明。
- 错误原因是**基于选项分布与题目结构的合理推断**，措辞用“可能原因/疑似”，不写成定论。
- 典型错误展示必须匿名化：移除姓名、考号、座位号、手写特征等可识别信息。
- 补偿练习不得照抄原试卷题目，应改写情境或数值；原题只在订正单中让学生订正本人作答的版本。
- 讲评稿是给教师课堂用的，语言面向讲解而不是面向学生答题；板书内容要具体可写。
- 数据只在工作区内处理，不发送到外部服务。

## 统计与算法

- **讲评价值公式**与默认权重：见 [讲评价值公式](references/review-formula.md)。
- **统计字段口径**（正确率 vs 得分率、知识点得分率聚合方式、分数段划分）：见 [统计口径](references/statistics.md)。

讲评价值公式中的“知识点重要程度”“后续学习影响”“错误代表性”缺项时使用默认权重；切换学科或年级时可由教师在配置中覆盖。

## 可选生成程序

当需要把分析结果导出为可直接打印/分发的文件时，使用内置生成程序把统计 JSON 转为讲评报告：

```bash
python3 scripts/shengcheng_jiangping_baogao.py 输入资料.json --output 班级讲评报告.html
```

输入格式见 [交付物结构](references/deliverables.md)。不要手工拼装报告页面，复用 `assets/report-template.html` 以保证版式一致。

## 页面能力（生成报告时）

生成的讲评报告页内置五个结果页签：班级概览、题目分析、知识点分析、讲评方案、补偿练习。题目分析中每题一张卡片（题号、正确率/得分率、高频错误、知识点、讲评建议），点击展开题目原文、答案分布、错误原因、典型错误、讲评稿、补偿练习。支持导出 Word/PDF。
