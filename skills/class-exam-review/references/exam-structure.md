# 试卷结构

解析试卷后每道题产出以下字段。字段命名与 [讲评报告 JSON](./deliverables.md#生成程序输入-json) 一致，供生成程序直接消费。

## 题目字段

| 字段 | 必填 | 类型 | 说明 |
| --- | --- | --- | --- |
| `id` | 是 | string | 题号，如 `"8"`、`"12"`、`"17-1"`。整卷内唯一。 |
| `type` | 是 | enum | `single` 单选 / `multiple` 多选 / `judge` 判断 / `subjective` 主观。第一版仅支持这四类。 |
| `stem` | 是 | string | 题干原文（可含公式、图表描述）。图表用 `[图：...]` 占位说明。 |
| `score` | 是 | number | 该题满分。 |
| `correct` | 客观题必填 | string \| string[] | 标准答案。单选/判断为单个值；多选为字母数组（顺序无关）。 |
| `rubric` | 主观题必填 | object | 评分点：`{ points: [{ desc, score }], total }`。 |
| `knowledge` | 否 | string[] | 关联知识点。资料未给出标为 `["待教师补充"]`。 |
| `difficulty` | 否 | enum | `easy / medium / hard`。缺省按 `medium` 处理并注明。 |
| `related` | 否 | string[] | 关联题号（例如第 12 题用到第 8 题结论）。 |
| `source` | 否 | string | 出处标注（教材单元/章节），便于定位复习。 |

## 抽取规则

- **题号识别**：按“题/N./第 N 题/(N)”等模式切分；切分失败时提示教师确认边界，不要自动编号。
- **题型判断**：以题干关键词（“单选/多选/判断/简答/计算/证明/作文”）为准；缺关键词时由选项结构推断（4 个字母选项→单选，多个正确项→多选）。
- **分值**：题末括号“（X 分）”或卷首分值说明表；两处冲突以题末为准。
- **知识点**：来自试卷标注则照录；未标注时不擅自归并，留空待补。
- **主观题评分点**：必须来自“评分标准”材料；材料只给总分无细则时，`rubric.points` 留空数组并标“待教师补充评分点”，不影响得分段统计。
- **公式与符号**：保留 LaTeX 或原始符号，不转写为白话；图表不识别精确数据，仅做文字描述占位。

## 教师确认清单

结构解析后、进入统计前，向教师展示一次清单：

- 题量、各题型数量与分值合计（核对是否等于卷面满分）；
- 知识点分布概览（哪些是“待补充”）；
- 存疑项（题号边界存疑、答案存疑、评分点缺失）。

教师确认或修正后才进入 [统计](./statistics.md)。
