# 输入格式

本技能接受两类输入：教师上传的**作答明细 Excel**（统计源），以及生成程序使用的**讲评报告 JSON**（由技能在分析完成后组装）。

## 一、作答明细 Excel（教师上传）

最低要求是一张宽表，每行一名学生，每题一列，含总分列。

### 推荐格式

| 姓名 | 第1题 | 第2题 | 第3题 | 第4题 | … | 总分 |
| --- | --- | --- | --- | --- | --- | --- |
| 王同学 | A | C | 4 | 8 | … | 78 |
| 李同学 | B | C | 2 | 6 | … | 65 |

### 字段口径

- **姓名列**：列名含“姓名/学生/学号”均可；技能识别后统一映射为 `studentId`。学号同样可识别。
- **题号列**：列名形如 `第1题 / Q1 / 题1 / 1`，技能按数字排序识别。客观题填选项（`A`/`AB`/`对`/`√` 等），主观题填该题得分（数字）。
- **客观题作答**：单选填单个字母；多选填多个字母（顺序无关，技能内部排序后比较）；判断题填 `对/错`、`T/F`、`√/×` 或 `正确/错误`，统一归一化。
- **主观题得分**：填数字（允许小数）。空缺、`缺考`、空白单元格视为未作答，不纳入该题分母。
- **总分列**：列名含“总分/合计/分数”均可；若存在则用于分数段分布，缺失时由各题得分求和回填并注明。
- **缺考标记**：整行全空或含“缺考”字样视为缺考，从全部分析中剔除，仅在概览中说明缺考人数。

### 容错

- 表头前后空行、合并单元格、多余说明行：自动跳过。
- 列顺序无关，按列名匹配。
- 同一学生重复出现：保留首次，重复行在日志中提示。
- 选项大小写、全半角、带括号（`(A)`）统一归一化后再比较。

### 不支持的格式

- 全班手写答题卡扫描件（第一版不识别，提示教师改交阅卷系统导出表）。
- 只有总分没有逐题得分的汇总表（无法做逐题分析）。
- 多工作表混合（仅读取第一个含“姓名”列的工作表，其余忽略并提示）。

## 二、讲评报告 JSON（生成程序输入）

技能完成分析后，把全部结果组装为 UTF-8 JSON 交给 `scripts/shengcheng_jiangping_baogao.py`。结构见 [交付物结构](./deliverables.md#生成程序输入-json)。生成程序会拒绝以下情况并报错：

- `meta` 或 `questions` 缺失；
- 题号重复或缺失；
- 任一题 `type` 不在 `single / multiple / judge / subjective` 之内；
- 客观题缺少 `correct` 与 `distribution`；
- 主观题缺少 `scoreBuckets` 与 `average`。
