# 讲评价值公式

用于把题目排出讲评顺序，避免按题号从头讲到尾。每道题算一个 `reviewValue`（越大越优先讲），再映射到四档。

## 公式

```
reviewValue =
    ( 错误人数      × W_people )
  × ( 知识点重要程度 × W_knowledge )
  × ( 后续学习影响  × W_downstream )
  × ( 错误代表性    × W_typical )
  ÷ ( 预计讲解时间  × W_time )
```

## 各因子取值

每个因子先归一到 `0.0–1.0`，乘以权重后参与运算。**所有因子必须可追溯到数据**，不得拍脑袋。

| 因子 | 取值依据 | 归一方式 |
| --- | --- | --- |
| 错误人数 | 由 [统计](./statistics.md) 得出错误人数 | `错误人数 ÷ 实考人数`（即 `1 - accuracy`） |
| 知识点重要程度 | 教师标注 / 课程标准核心概念 | 核心=`1.0`、重要=`0.7`、一般=`0.4`、待补=`0.5`（默认） |
| 后续学习影响 | 该知识点在本单元/后续单元是否反复出现 | 高=`1.0`、中=`0.6`、低=`0.3`、待补=`0.5` |
| 错误代表性 | 高频错误是否反映典型 misconceptions（看 `topWrong` 占比） | 主错项占比 `≥50%`=1.0、`30–50%`=0.7、`10–30%`=0.4、`<10%`=0.2 |
| 预计讲解时间 | 题型与难度估算（分钟） | `min(预计时间, 8) ÷ 8`（封顶 8 分钟） |

## 默认权重

| 权重 | 默认值 | 说明 |
| --- | --- | --- |
| `W_people` | 1.0 | 错误人数最硬，权重最大 |
| `W_knowledge` | 0.9 | 知识点重要程度 |
| `W_downstream` | 0.7 | 对后续的影响 |
| `W_typical` | 0.6 | 错误是否典型 |
| `W_time` | 0.5 | 单位时间收益 |

权重可由教师在配置中按学科/年级覆盖；覆盖后须在结果中注明使用了非默认权重。

## 分档阈值

按整卷 `reviewValue` 从高到低排序后分四档（不是按绝对值，而是按分布）：

| 档位 | 占比 | 处理 |
| --- | --- | --- |
| **必须重点讲** | 前 ~20%（至少 1 题） | 写完整讲评稿（见 [讲评稿模板](./review-script.md)） |
| **简要讲解** | 接下来 ~30% | 给一句话讲评要点 + 板书关键词 |
| **学生自行订正** | 接下来 ~35% | 只在订正单中点名，不在课堂统一讲 |
| **无需统一讲解** | 剩余 ~15% | 跳过（含正确率很高或只有个别人错的题） |

> 题量很少（< 10 题）时，分档可退化为“重点讲 / 简要讲 / 跳过”三档，并在结果中说明。

## 边界情况

- **错误人数相同、价值并列**：并列时按题号升序排，不随意打破。
- **正确率 100% 但知识点重要**：仍然进“无需统一讲解”，但可列入“班级掌握良好”清单。
- **零人错的题**：跳过讲评计算，直接归入“无需统一讲解”。
- **主观题**：用 `1 - scoreRate` 替代“错误人数”，其余因子同上。
- **预计讲解时间**：客观题默认 2 分钟，主观题按难度 `easy=3 / medium=5 / hard=7` 分钟；教师可在确认结构时手改。

## 透明性

输出 `reviewPlan` 时，每道题须附上 `reviewValue` 的**各分项原始值**（错误人数、归一化后的 5 个因子、最终得分），让教师能看出为什么某题被排到重点讲。不接受只给排序不给分项的黑盒结果。
