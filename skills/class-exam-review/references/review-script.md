# 讲评稿模板

每道“必须重点讲”的题目产出一份可直接走进课堂的讲评稿。语言面向教师讲解，不面向学生答题。

## 模板

```
第{题号}题（{题型}，{满分}分；知识点：{知识点列表}）

【班级表现】
正确率/得分率：{xx}%；高频错误：{选项或错误类型}（{占比}%）。
（主观题：得分段分布、零分率。）

【讲评目标】
一句话说清这堂课讲完这题后学生应当能区分/识别/做到什么。
（例：区分函数零点与方程根的关系。）

【建议讲评过程】
1. 先问学生题目要求判断什么（不直接给答案）。
2. 展示主错项 {X} 的典型思路，让学生自己发现漏洞。
3. 指出该思路遗漏的条件（定义域/隐含条件/边界）。
4. 回到正确方法，规范书写一遍。
5. 总结同类题的判断顺序 / 易错点。

【板书内容】
- 主错思路 → 漏洞点（具体写）
- 正确思路 → 关键步骤（具体写）
- 易错点小结：……

【衔接补偿练习】
配套：补偿练习第 {n} 题（同类巩固）。
```

## 规则

- **班级表现**数字必须来自 [统计](./statistics.md)，不重新估算。
- **讲评目标**必须是可观测的学生行为（“能区分/能识别/能写出”），不是教师任务。
- **讲评过程**至少 4 步，第一步尽量是“让学生先说/先判断”，避免直接讲授。
- **板书内容**要具体到能照抄上黑板，不留“……”占位（除非是同类题模板部分）。
- **主观题**讲评过程增加“展示一份匿名典型卷 → 师生共评 → 对照评分点”环节。
- 配套的补偿练习题号必须与 [交付物](./deliverables.md) 中的 `remedialExercises` 对应。
