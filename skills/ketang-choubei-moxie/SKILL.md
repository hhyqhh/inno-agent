# 课堂抽背与默写

将教学资料整理为经过教师确认的题目，再使用内置生成程序制作一个无需联网、适合投屏和打印的 HTML。

## 工作流程

1. 读取用户提供的资料。支持直接粘贴的文字，以及系统能够读取的 PDF、Word、表格、图片或纯文本文件。
2. 判断资料类型并整理题目：
   - 古诗：上句接下句、下句接上句、全文背诵、挖空默写。
   - 文言文：段落背诵、重点句默写、翻译、重点字词解释。
   - 英语单词：英译中、中译英、拼写默写。
   - 政治或普通问答：题目、答题关键词、参考答案。
   - 普通课文：按自然句或自然段截取，不在一句话中间截断。
3. 忽略页眉、页脚、页码、目录、作者简介和与抽查无关的注释，除非用户要求保留。不得补写资料中不存在的原文或答案；无法确认的内容标为待确认。
4. 读取学生名单。去掉空行和完全重复的姓名，保留原顺序。不要在回复中不必要地重复完整名单。
5. 向教师展示简短的确认清单：活动标题、题目总数、各题型数量、学生人数、随机规则。列出待确认内容。教师确认前不得生成最终页面。
6. 按 [输入格式](references/input-schema.md) 创建 UTF-8 JSON。默认采用“一轮不重复”、5 秒倒计时、答案默认隐藏、音效关闭。
7. 运行：

   ```bash
   python3 scripts/shengcheng_ketang_html.py 输入资料.json --output 课堂抽背.html
   ```

8. 确认程序成功，检查输出文件存在，并在交付时说明题目数、学生数和文件位置。

## 内容质量规则

- 原文、题目和答案必须能够追溯到用户资料；常识性解释也要与原资料区分。
- 古诗文保留原有标点和段落。异体字、通假字或教材版本差异不得擅自统一。
- 思考题的“关键词提示”和“参考答案”分开；投屏时先显示题目，答案由教师手动展开。
- 单词题保留大小写和必要的词性信息，但不要把例句误当作答案。
- 每题都设置稳定且不重复的 `id`，例如 `poem-001`、`word-003`。
- 学生名单为空时仍可生成纯抽题与默写页面。
- 学生姓名只写入本地 HTML，不发送到其他服务。提醒教师妥善保管含真实姓名的文件。

## 页面能力

生成页面内置：随机学生、随机题目、学生题目同时抽取、一轮不重复、临时排除学生、倒计时、答案与提示显示、掌握情况标记、本地进度保存、重置本轮、全屏和题目卷/答案卷打印。

不要手工重写页面代码。复用 `assets/classroom-template.html`，通过生成程序注入数据，以保持功能和版式一致。
